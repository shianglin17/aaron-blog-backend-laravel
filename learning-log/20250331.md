# Laravel 日期處理最佳實踐

## 時間：2025-03-31 15:00

### 主題：Model 日期欄位處理

#### 問題描述
在使用 Laravel Resource 處理日期時遇到 "Call to a member function format() on string" 錯誤，這是因為日期欄位沒有被正確轉換為 Carbon 實例。

#### 解決方案
1. 使用 `$casts` 而不是 `$dates`
```php
protected $casts = [
    'published_at' => 'datetime',
    'created_at' => 'datetime',
    'updated_at' => 'datetime',
];
```

2. 在 Resource 中安全地處理日期
```php
'created_at' => $this->created_at?->format('Y-m-d H:i:s'),
```

#### 最佳實踐
1. 總是使用 `$casts` 來定義日期欄位，這是 Laravel 推薦的方式
2. 使用空值運算符 `?->` 來安全地處理可能為 null 的日期
3. 明確定義所有需要日期轉換的欄位，包括 `created_at` 和 `updated_at`

#### 相關文檔
- [Laravel 文檔 - Eloquent: Mutators & Casting](https://laravel.com/docs/10.x/eloquent-mutators)
- [Laravel 文檔 - Eloquent Resources](https://laravel.com/docs/10.x/eloquent-resources)

#### 注意事項
- `$dates` 屬性雖然仍然可用，但在新版本中推薦使用 `$casts`
- 使用 `$casts` 可以更靈活地定義轉換類型，不限於日期
- 在處理 API 響應時，確保日期格式統一很重要

# Laravel API 認證方案比較研究

## 時間：2025-03-31 16:00

### 主題：API 認證方案選擇

#### 可用方案比較

1. **Laravel Sanctum**
   - 優點：
     - 輕量級解決方案
     - 完美整合 Laravel
     - 支援 SPA 和 API 認證
     - 內建 CSRF 保護
     - 官方維護
   - 適用場景：
     - 單頁應用（SPA）
     - 簡單的 API 認證需求
     - 需要 session 支援的應用

2. **Laravel Passport**
   - 優點：
     - 完整的 OAuth2 實現
     - 支援多種授權方式
     - 功能豐富
   - 適用場景：
     - 需要 OAuth2 的專案
     - 第三方應用授權
     - 複雜的權限管理

3. **JWT（JSON Web Tokens）**
   - 優點：
     - 無狀態認證
     - 跨平台支援
     - 效能較好
   - 適用場景：
     - 分散式系統
     - 微服務架構
     - 純 API 專案

#### 建議方案
對於部落格後台管理系統，建議使用 **Laravel Sanctum**，原因如下：
1. 簡單易用：設置簡單，維護成本低
2. 安全性高：內建多種安全機制
3. 足夠輕量：不會增加不必要的複雜度
4. 完整整合：與 Laravel 生態系統完美配合

#### 實作步驟
1. 安裝 Sanctum：
```bash
composer require laravel/sanctum
php artisan vendor:publish --provider="Laravel\Sanctum\SanctumServiceProvider"
php artisan migrate
```

2. 配置 User Model：
```php
use Laravel\Sanctum\HasApiTokens;

class User extends Authenticatable
{
    use HasApiTokens, HasFactory, Notifiable;
}
```

3. 配置 Kernel.php：
```php
'api' => [
    \Laravel\Sanctum\Http\Middleware\EnsureFrontendRequestsAreStateful::class,
    'throttle:api',
    \Illuminate\Routing\Middleware\SubstituteBindings::class,
],
```

#### 實作步驟

1. **環境準備**
```bash
# 安裝 JWT 套件
composer require php-open-source-saver/jwt-auth

# 發布配置文件
php artisan vendor:publish --provider="PHPOpenSourceSaver\JWTAuth\Providers\LaravelServiceProvider"

# 生成 JWT 密鑰
php artisan jwt:secret
```

2. **基礎配置**
   - 配置 `config/auth.php`：
```php
'guards' => [
    'api' => [
        'driver' => 'jwt',
        'provider' => 'users',
    ],
],
```

3. **更新 User Model**
```php
use PHPOpenSourceSaver\JWTAuth\Contracts\JWTSubject;

class User extends Authenticatable implements JWTSubject
{
    // JWT 必要方法
    public function getJWTIdentifier()
    {
        return $this->getKey();
    }

    public function getJWTCustomClaims()
    {
        return [];
    }
}
```

4. **建立認證控制器**
```php
namespace App\Http\Controllers;

use Illuminate\Http\Request;
use Illuminate\Support\Facades\Auth;

class AuthController extends Controller
{
    public function login(Request $request)
    {
        $credentials = $request->validate([
            'email' => 'required|email',
            'password' => 'required',
        ]);

        if (!$token = auth('api')->attempt($credentials)) {
            return response()->json(['error' => '未授權'], 401);
        }

        return $this->respondWithToken($token);
    }

    protected function respondWithToken($token)
    {
        return response()->json([
            'access_token' => $token,
            'token_type' => 'bearer',
            'expires_in' => auth('api')->factory()->getTTL() * 60
        ]);
    }
}
```

5. **設置路由**
```php
Route::group(['prefix' => 'api'], function () {
    Route::post('login', [AuthController::class, 'login']);
    
    Route::group(['middleware' => 'auth:api'], function () {
        Route::post('logout', [AuthController::class, 'logout']);
        Route::post('refresh', [AuthController::class, 'refresh']);
        Route::get('me', [AuthController::class, 'me']);
    });
});
```

#### 進階功能實作

1. **Token 刷新機制**
```php
public function refresh()
{
    return $this->respondWithToken(auth('api')->refresh());
}
```

2. **自動刷新中間件**
```php
namespace App\Http\Middleware;

use Closure;
use Illuminate\Http\Request;
use Symfony\Component\HttpFoundation\Response;

class RefreshToken
{
    public function handle(Request $request, Closure $next): Response
    {
        $response = $next($request);
        
        // 如果 token 即將過期，自動刷新
        if (auth('api')->check() && auth('api')->payload()->get('exp') - time() < 3600) {
            $token = auth('api')->refresh();
            $response->headers->set('Authorization', 'Bearer ' . $token);
        }
        
        return $response;
    }
}
```
